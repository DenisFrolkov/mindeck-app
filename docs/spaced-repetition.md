# Spaced Repetition System

Алгоритм основан на **SM-2** (Piotr Wozniak, 1990).
Источник: https://www.super-memory.com/english/ol/sm2.htm

Цель — показывать карточку в нужный момент: достаточно редко, чтобы она успела «забыться», но достаточно часто, чтобы не забылась совсем.

---

## Состояния карточки

```
NEW → LEARNING → REVIEW
                   ↓ (Again)
                 LAPSE → LEARNING
```

| Состояние | Описание |
|-----------|----------|
| **NEW** | Карточка ещё ни разу не показывалась |
| **LEARNING** | Карточка в фазе обучения, короткие интервалы |
| **REVIEW** | Карточка выучена, длинные интервалы (дни/недели/месяцы) |
| **LAPSE** | Карточка была в REVIEW, но пользователь её забыл — возвращается в LEARNING |

---

## Поля карточки

| Поле | Тип | Описание |
|------|-----|----------|
| `cardState` | `CardState` | Текущее состояние: NEW / LEARNING / REVIEW / LAPSE |
| `easeFactor` | `Float` | Коэффициент лёгкости. Начальное значение: **2.5**, минимум: **1.3** |
| `interval` | `Int` | Текущий интервал в днях (актуален только для REVIEW) |
| `learningStep` | `Int` | Текущий шаг в фазе обучения (0 или 1) |
| `nextReviewDate` | `Long` | Unix timestamp следующего показа |
| `repetitionCount` | `Int` | Общее количество повторений |
| `lapseCount` | `Int` | Сколько раз карточка упала из REVIEW обратно в LEARNING |
| `firstReviewDate` | `Long?` | Дата первого показа (для статистики и дневного лимита) |
| `lastReviewDate` | `Long?` | Дата последнего показа (для статистики) |

---

## Кнопки оценки

Оригинальный SM-2 использует шкалу качества **q от 0 до 5**. Четыре кнопки маппятся так:

| Кнопка | q | Смысл |
|--------|---|-------|
| **Again** | 1 | Не знаю / забыл |
| **Hard** | 3 | Знаю, но с трудом |
| **Good** | 4 | Знаю нормально |
| **Easy** | 5 | Знаю отлично |

---

## Формула изменения easeFactor

Из оригинального SM-2:

```
EF = EF + (0.1 - (5 - q) × (0.08 + (5 - q) × 0.02))
```

Результат для каждой кнопки:

| Кнопка | q | Изменение EF |
|--------|---|--------------|
| **Again** | 1 | −0.54 |
| **Hard** | 3 | −0.14 |
| **Good** | 4 | 0.00 |
| **Easy** | 5 | +0.10 |

`easeFactor` не может опуститься ниже **1.3**.

---

## Фаза Learning / Lapse

Шаги: **1 мин → 10 мин**

| Кнопка | Действие |
|--------|----------|
| **Again** | Сброс на шаг 0 → `nextReview = now + 1 мин` |
| **Hard** | Остаться на текущем шаге → `nextReview = now + текущий шаг` |
| **Good** | Перейти на следующий шаг. Если шагов больше нет → выпустить в REVIEW с `interval = 1` |
| **Easy** | Сразу выпустить в REVIEW с `interval = 4` |

---

## Фаза Review

Интервалы по оригинальному SM-2:
- 1-й REVIEW: `interval = 1 день`
- 2-й REVIEW: `interval = 6 дней`
- Последующие: `interval = предыдущий interval × EF`

| Кнопка | Новый interval | Изменение EF |
|--------|---------------|--------------|
| **Again** | Сброс в LAPSE → Learning шаг 0 | −0.54 |
| **Hard** | `interval × 1.2` | −0.14 |
| **Good** | `interval × EF` | 0.00 |
| **Easy** | `interval × EF × 1.3` | +0.10 |

> **Примечание:** в оригинальном SM-2 все прошедшие оценки (q ≥ 3) используют один множитель `interval × EF` — разница только в изменении EF. Множители ×1.2 (Hard) и ×1.3 (Easy) — адаптация из Anki, которая делает разницу между кнопками ощутимой уже в текущем повторении, а не только в будущих.

После расчёта: `nextReviewDate = now + interval (в днях)`.

---

## Внутрисессионный повтор

Карточка остаётся в очереди сессии, если после оценки её состояние — **LEARNING** или **LAPSE**. Как только карточка переходит в **REVIEW** — она уходит из сессии до следующего дня.

| Сценарий | Результат |
|----------|-----------|
| `Again` на NEW/LEARNING/LAPSE | → LAPSE/LEARNING → **остаётся** в сессии |
| `Hard` на LEARNING/LAPSE | → остаётся в LEARNING → **остаётся** в сессии |
| `Good`/`Easy` на LEARNING | → переходит в REVIEW → **уходит** из сессии |
| `Hard` на REVIEW | → остаётся в REVIEW → **уходит** из сессии |
| `Again` на REVIEW | → переходит в LAPSE → **остаётся** в сессии |

Это отклонение от оригинального SM-2: там Hard всегда оставляет карточку в сессии. Здесь решение принимается по итоговому состоянию, а не по нажатой кнопке — это позволяет не «гонять» REVIEW-карточку внутри сессии при нажатии Hard, когда она всё равно не требует немедленного закрепления.

---

## Порядок показа карточек

Сессия формируется в следующем порядке:

1. **LEARNING + LAPSE** — у которых `nextReviewDate ≤ now`
2. **REVIEW** — у которых `nextReviewDate ≤ now`
3. **NEW** — до дневного лимита

Карточки из п.1 и п.2 всегда показываются полностью — пропускать нельзя.

---

## Дневной лимит новых карточек

Максимум **20 новых карточек в день**.

Счёт ведётся по `firstReviewDate`: если карточка получила первый показ сегодня — она засчитывается в лимит. Повторения (REVIEW / LAPSE) лимитом не ограничены.

> В будущем лимит выносится в настройки пользователя.
